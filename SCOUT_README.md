## Data Architecture
- **players.json** — ~10k players. id, name, team, positions, level, age, rank, mlbam_id, fangraphs_id, birthDate, model_scores. Never wipe. `rank` written by compute engine. `model_scores` written by build-scores.js.
- **db.json** — leagues, teams, rosters. Safe to delete + re-sync.
- **stats.json** — current season (2026) stats keyed by Fantrax ID. Always wiped on sync.
- **rankings/sources/** — one JSON per imported ranking source. Never auto-deleted.
- **rankings/rankings.json** — computed consensus output. Rewritten on each compute.
- **history/YYYY.json** — all player stat lines for that year keyed by mlbam_id. Contains both MLB and MiLB lines. Re-run scripts to add new players — resume-capable. 2026.json exists and is updated by stats sync, not the history scripts.
- **model/norms.json** — level+league+year stat norms. Primary key: `"AA|2023"`. Rebuild by running `build-norms.js`.
- **model/mlb-scores.json** — per-season MLB fantasy point rates. Rebuild by running `build-mlb-scores.js`.
- **model/mlb-tools.json** — per-player career MLB tool grades keyed by mlbam_id. **Source of truth for MLB vs minors designation** — presence = graduated. Includes `_seasons` and `_pa`/`_ip` for blend logic. Rebuild by running `build-mlb-tools.py`.
- **model/age-elasticity.json** — empirically derived k coefficients per level × stat. Built by `build-age-elasticity.py` but no longer used in scoring — age adjustment now applied at prediction level in build-scores.js. Kept for reference.
- **model/regression.json** — per-tool blended level model. Each level has slope, intercept, corr, n. Rebuild by running `build-regression.py`.
- **model/hot-sheet.json** — top 20 bats + 20 arms by current season model delta. Written by build-scores.js every run.
- **model/scores-snapshot.json** — snapshot of model scores (overall + _sample) keyed by Fantrax ID. Written as a comparison baseline.

## Hot Sheet Page (`/hot-sheet`)
Biggest model score gainers this season — top 20 bats and 20 arms.

### How it works:
- Generated by build-scores.js as part of every model run
- For each prospect: compute normalized overall WITH current year data vs WITHOUT current year data
- Delta = with - without. Sorted by delta descending.
- Uses same normalization pool as the model — no separate scoring pass needed
- Filters: overall ≥ 100, career sample ≥ 150 PA (hitters) / 60 IP (pitchers), delta > 0
- Columns: ΔOVR · POS · PLAYER · LEV · tool grades · OVR+
- **Stat line below player name:** bats show OPS · K% · BB% · HR · SB; arms show W-L · IP · ERA · K% · BB%
- Gold highlight + left border for players owned by Winston Salem Dash
- League dots show ownership across all 3 leagues

### Hot sheet math:
- `overallWith` = model_scores.overall (already computed)
- `overallWithout` = re-normalize tools using (wSum - cySum) / (wTot - cyTot) per tool, then composite
- Both use identical toolVals normalization — apples to apples
- Early season: small deltas (1-3 pts) are correct and expected — weights handle sample size naturally

## Ranking System
Three source types feed the consensus engine:

### Source Types
- **Overall (O)** — dynasty overall rankings (Clegg, Fantrax, PLive Top 1000)
- **Prospect (P)** — prospect-only rankings (Clegg Prospects, Fantrax Prospects, PLive Top 500)
- **Open Universe (X)** — unranked prospects with tier signals (PLive Open Universe). Tier format: "Top 30" → rank 30, second in tier → 31, etc.

### Import Flow
Rankings page → Add Source → pick type → map columns → Save Source.
- Tier format option: converts "Top 30" style columns to rank numbers
- Column mapping UI auto-detects common column names
- Sources saved as `data/rankings/sources/YYYYMMDD_sourcename.json`

### Compute Engine (`/api/rankings/compute`)
1. **Overall consensus** — weighted average of O sources per player. Staleness weight: linear decay, 100% at day 0, 0% at day 365. NR penalty = max rank in that source + 1. Competition ranking on final avg.
2. **Full prospect ranking** — all P source players ranked by weighted avg (includes O players, used only for anchor lookup).
3. **Prospect-only ranking** — same but excludes O players. These slot after the last O rank in the final E list.
4. **Open universe slotting** — X players not in O or P. Their tier rank (e.g. 10) = target position in the full prospect ranking. Find that P player → look up their O rank → insert X player right after them in E. Preserves prospect ordering even when O ranks are non-monotonic by using running max sort key.
5. **Final E list** — O + anchored X interleaved, competition ranked. Then P-only + fallback X appended after.
6. Writes `rankings.json` and patches `rank` field in `players.json`.

### Name matching
Exact name → position tiebreak → team tiebreak → if still ambiguous both players get the better rank. Diacritics stripped for fuzzy matching.

## Proprietary Ranking Model (`data/model/`)
Goal: predict MLB tool grades from MiLB history to power dynasty prospect valuation.

**Model rebuilt Apr 2026** — full rebuild with correct architecture. Prior version had fabricated R² scores, broken pitcher regression, hardcoded age multipliers, and phantom DSL data.

### Tool definitions (wRC+-style, 100 = prospect pool avg after normalization):
- **Hit+** — k% (inverted) + bb%, blended. Hitters only.
- **Power+** — iso. Hitters only.
- **Speed+** — SB / (H + BB + HBP) = SB rate per time on base. Hitters only. Best predictor (R²=0.537).
- **Stuff+** — k% vs bf. Pitchers only.
- **Control+** — bb% vs bf. Pitchers only.

### Composite weights (derived from fantasy value correlation analysis, Apr 2026):
- **Hitters:** hit=0.42, power=0.47, speed=0.11
- **Pitchers:** stuff=0.70, control=0.30

### Shrinkage (sample confidence):
- Each **tool** shrunk individually toward 100: `confidence = sample / (sample + K)`
- K=200 PA (hitters), K=80 IP (pitchers)
- **Overall = pure weighted blend of shrunk tools — no second shrinkage applied**
- This means overall is always visibly the weighted average of what you see in the tool columns
- `_raw` stored per tool (pre-shrinkage normalized value) for hover tooltips — ceiling/future grade
- `_sample` and `_confidence` stored on model_scores for transparency

### Recency decay (added Apr 2026):
- Each season's weight multiplied by `0.75^(currentYear - season)`
- 2026 = 100%, 2025 = 75%, 2024 = 56%, 2023 = 42%, 2018 = ~18%
- Prevents old data from dominating; buy-low injured guys still show up if recent data exists

### Recency requirement:
- Player must have at least one qualifying MiLB season in the last 3 years to receive a score
- Uses `new Date().getFullYear() - 3` — dynamic, not hardcoded

### Age penalty (added Apr 2026):
- Applied **inside scoreTool**, per season per level — same loop as elasticity
- Addresses survivor bias: regression training set only includes players who reached MLB, so quad-A guys who never made it are underrepresented
- Formula: `yearsOver = max(0, age - AVG_AGES[level])` for that season's level
- `agePenalty = max(0.5, 1 - 0.04 × yearsOver²)` — quadratic, capped at 50% discount
- Applied to regression pred before it enters weighted sum: `pred = rawPred × agePenalty`
- Orthogonal to elasticity: elasticity boosts young-for-level seasons, penalty discounts old-for-level seasons
- Examples at AAA (avg 26.5): age 27 → 1% discount, age 28 → 9%, age 29 → 25%, age 30 → 49%
- Hit+ is most affected — k%/bb% age-elasticity at AAA is near zero (k=0.0086) so elasticity alone wasn't punishing quad-A guys

### Pipeline (auto-runs after every Sync Stats):
```bash

python3 ~/Desktop/fantasy-baseball/scripts/build-mlb-tools.py
python3 ~/Desktop/fantasy-baseball/scripts/build-regression.py
```

### Stage 1: Level Norms (`build-norms.js` → `norms.json`)
Computes mean and stdev for every stat by level+year (2015–2026).
- Min thresholds: 50 PA (hitters), 20 IP (pitchers)
- Primary key: `"AA|2023"`

### Stage 2: MLB Fantasy Rates (`build-mlb-scores.js` → `mlb-scores.json`)
Not in auto-run pipeline. Run manually when scoring formula changes.
- Hitter scoring: H×1 + 2B×1 + 3B×2 + HR×5 + RBI×1 + R×1 + BB×1 + SB×4 + HBP×1 + TB×0.25 + CS×(−4) + SO×(−1)
- Pitcher scoring: IP×2 + K×1 + W×10 + ER×(−1) + H×(−1) + BB×(−1) + L×(−10)

### Stage 3: MLB Tool Grades (`build-mlb-tools.py` → `mlb-tools.json`)
- Qualifying: MIN_SEASONS=1, 100+ PA/season (hitters), 50+ IP + 5+ GS/season (pitchers SP only)
- Per season: z-score stats against `MLB|YYYY` norms, weight by PA/IP
- Roll up to career weighted-average z-scores → convert to +scale (100=MLB avg, 15pts=1SD)
- **Presence in this file = graduated/MLB for the entire app.**
- `_seasons` field used by UI for tweener blend logic: `_seasons >= 2` = fully graduated, `_seasons === 1` = blend with MiLB model_scores
- Tweener blend: `mlbWeight = min(1, _pa/200)` for hitters, `min(1, _ip/100)` for pitchers

### Stage 4: Age Elasticity (`build-age-elasticity.py` → `age-elasticity.json`)
Not in auto-run pipeline. Run manually — doesn't need real-time updates.
- Derives empirical k coefficient per level × stat for `exp(k × age_diff)` multiplier
- Key findings: ISO most age-elastic. K% and BB% barely age-elastic. DSL k=0.

### Stage 5: Regression (`build-regression.py` → `regression.json`)
Per-tool blended level model.
- For each tool × level × stat: fit `outcome = slope × z_adj + intercept` on training players
- Training players: those with both MiLB history (2015+) and qualifying MLB tool grades
- Scoring: `score = Σ(slope × z_adj × corr × sample × recencyDecay) / Σ(corr × sample × recencyDecay)`
- Validated R²: Hit+=0.169, Power+=0.272, Speed+=0.537, Stuff+=0.301, Control+=0.195

### Stage 6: Score Prospects (`build-scores.js`)
- Scores all rookie-eligible players (not in mlb-tools.json)
- No minimum sample threshold — weights handle small samples naturally
- Recency filter: must have qualifying MiLB data within last 3 years
- Recency decay: `weight × 0.75^(currentYear - season)`
- **Scoring flow per tool:** raw stat z-scores → regression pred → age boost on pred (`100 + (pred-100) × ageW`) → weighted sum → normalize pool-wide to 100±15 → shrink individually toward 100
- **Overall:** pure weighted blend of shrunk tools, no additional shrinkage
- model_scores fields: hit/power/speed (hitters) or stuff/control (pitchers), overall, _raw (pre-shrink per tool), _sample, _confidence
- Also writes hot-sheet.json: top 20 bats + 20 arms by current season contribution delta
- Also writes scores-snapshot.json: point-in-time overall + _sample per player (for comparison/debugging)
- ~6,683 scored (Apr 2026)

### Hot Sheet generation (inside build-scores.js):
scoreTool returns wSum/wTot/cySum/cyTot per tool (full career weighted sums + current year subset).
After normalization, for each player:
- `overallWith` = model_scores.overall
- `overallWithout` = normalize((wSum-cySum)/(wTot-cyTot)) per tool → composite
- `delta` = overallWith - overallWithout
Filters: overall ≥ 100, career sample ≥ 150 PA / 60 IP, delta > 0, hasCY = true

## Player ID Deduplication

**Type 1: Ghost duplicates** — one entry has real team, other has `(N/A)`. Auto-fixed by `/tmp/fix_dupes.js` — keeps real entry, nulls ghost's mlbam_id.

**Type 2: Name collision mismatches** — two real different players incorrectly linked to same mlbam_id. Fixed by hitting MLB API for each dupe and matching on current team/position. ~41 cleared (Apr 2026).
